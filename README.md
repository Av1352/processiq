# ProcessIQ

AI-powered process intelligence for business event logs. Upload an order-to-cash CSV, get bottleneck detection, rework analysis, dropout rates, and a plain-language executive summary generated by Claude.

**Live:** [processiq.vercel.app](https://processiq.vercel.app)

---

## What it does

Upload any business process event log. ProcessIQ gives you four things:

1. **Throughput metrics** — average end-to-end time, case volume, completion rates
2. **Bottleneck detection** — activities where average wait time exceeds 2x the median
3. **Rework and dropout analysis** — cases with repeated activities or incomplete flows
4. **AI executive summary** — Claude translates the numbers into plain business language a CFO can act on in 30 seconds

No dashboards to configure. No SQL. Upload a CSV and get answers.

## Event Log Format

Your CSV needs these columns:

| Column | Description | Example |
|--------|-------------|---------|
| `case_id` | Unique process instance | `ORD-001` |
| `activity` | Process step name | `Credit Check` |
| `timestamp` | Event datetime | `2024-01-15 09:30:00` |
| `resource` | Team or person | `Finance Team` |
| `amount` | Transaction value | `25000` |

A sample order-to-cash log with 500 cases is included. Run `python generate_data.py` to regenerate it with realistic patterns including bottlenecks, rework, and dropouts.

## Run Locally

**Backend**
```bash
cd backend
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
pip install -r requirements.txt
python generate_data.py
cp .env.example .env         # add ANTHROPIC_API_KEY
python main.py
```

**Frontend**
```bash
cd frontend
npm install
cp .env.example .env         # set VITE_API_URL=http://localhost:8000
npm run dev
```

Open `http://localhost:5173` and click "Load Sample Data" to test the full pipeline.

## Analysis Engine

For each uploaded log, the engine calculates:

- **Throughput time** — days from first to last event per case
- **Activity wait time** — average days between each consecutive activity pair
- **Bottleneck detection** — activities with wait time more than 2x the median
- **Rework rate** — percentage of cases where any activity appears more than once
- **Dropout rate** — percentage of cases that never reach the final activity
- **Resource workload** — case count per resource, sorted by load

All metrics feed into the Claude prompt for the executive summary.
